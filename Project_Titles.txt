Project Title
Unified Network Operations & Automation Platform
Project Overview

Develop a Python-based Network Automation Platform that helps network engineers perform day-to-day operational tasks from a single application. The platform should automate inventory collection, configuration backup, health monitoring, API-based management, reporting, and alerting using modern network automation techniques.

Module 1: Device Inventory Management
Description

Create a module that connects to network devices and collects basic inventory information.

Responsibilities
Collect hostname
Collect model number
Collect OS version
Collect serial number
Maintain device inventory
Deliverable

Inventory report containing all managed devices.

Module 2: Configuration Backup Management
Description

Develop a module that automatically retrieves and stores device configurations.

Responsibilities
Collect running configuration
Store backups locally
Maintain backup history
Compare current and previous backups
Deliverable

Configuration backup repository and backup status report.

Module 3: Network Health Monitoring
Description

Build a monitoring system that continuously checks device health and availability.

Responsibilities
Monitor CPU utilization
Monitor memory utilization
Monitor interface status
Check device reachability
Detect threshold violations
Deliverable

Device health report showing current network status.

Module 4: REST API Integration
Description

Implement network automation using REST APIs to retrieve and manage device information.

Responsibilities
Authenticate with API services
Retrieve device information
Parse JSON responses
Display collected information
Deliverable

API-generated device information report.

Module 5: RESTCONF Management
Description

Develop a RESTCONF-based module for managing Cisco IOS-XE devices.

Responsibilities
Retrieve hostname
Retrieve interface information
Retrieve IP configuration
Access YANG-based data models
Deliverable

RESTCONF operational data report.

Module 6: NETCONF Management
Description

Implement device management using NETCONF and YANG models.

Responsibilities
Retrieve configuration data
Retrieve operational data
Modify configurations
Validate configuration changes
Deliverable

NETCONF configuration management report.

Module 7: Multi-Vendor Device Support
Description

Extend the platform to support multiple network vendors.

Responsibilities
Manage Cisco devices
Manage Juniper devices
Standardize device access
Collect common operational data
Deliverable

Multi-vendor inventory and status report.

Module 8: Automated Reporting System
Description

Generate management and operational reports using collected network data.

Responsibilities
Generate inventory reports
Generate health reports
Generate backup reports
Export reports in CSV/Excel format
Deliverable

Consolidated network operations report.

Module 9: Alert & Notification System
Description

Notify administrators when network issues are detected.

Responsibilities
Monitor thresholds
Detect failures
Generate alerts
Send email notifications
Deliverable

Alert notification log and email alerts.

Module 10: Troubleshooting & Diagnostics
Description

Automate the collection of troubleshooting information during network incidents.

Responsibilities
Collect interface details
Collect routing information
Collect logs
Collect device resource usage
Generate diagnostic reports
Deliverable

Network troubleshooting report with identified issues.

Expected Outcome

At the end of the project, the platform should allow a network engineer to:

View network inventory
Backup configurations
Monitor device health
Access devices through REST APIs
Manage devices using RESTCONF
Manage devices using NETCONF
Generate reports
Receive alerts
Collect troubleshooting information

This project is designed to cover all major concepts discussed during the 11-day Network Automation training program and serves as the final assessment project.
